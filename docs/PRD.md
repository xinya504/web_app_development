# PRD - 隨機抽號系統

## 1. 專案概述
- **背景與動機**：在舉辦活動、課堂點名或抽獎時，常常需要一個公平且隨機的抽號機制。為了解決手動抽籤的不便，我們需要一個自動化的隨機抽號系統。
- **目標用戶**：需要隨機抽號的人（如活動主辦方、教師、需要做隨機決策的個人等）。
- **核心價值主張**：提供簡單、快速、可自訂的隨機抽號體驗，並具備帳號管理與紀錄保存功能，確保抽號過程公平公開。

## 2. 功能需求
主要功能：
1. **隨機抽取號碼**：系統能根據設定的範圍隨機產生一個或多個號碼。
2. **讓使用者輸入選項**：除了純數字號碼外，使用者可以輸入自訂的名單或選項進行隨機抽取。
3. **可以登入系統**：使用者可註冊與登入帳號，管理自己的抽號設定。
4. **歷史紀錄儲存與查詢**：登入的使用者可以儲存並查看過去的抽號紀錄。
5. **抽籤結果分享**：提供簡單的介面將抽號結果匯出或產生連結分享給其他人。

**使用者故事 (User Stories)**：
- 作為 **活動主辦方 (使用者)**，我希望 **能設定抽籤的數字範圍並進行隨機抽取號碼**，以便 **公平地抽出中獎者**。
- 作為 **教師 (使用者)**，我希望 **能輸入學生的姓名清單作為選項進行抽取**，以便 **在課堂上隨機點名回答問題**。
- 作為 **一般使用者**，我希望 **能夠註冊並登入系統**，以便 **保存我常用的抽籤名單或設定**。
- 作為 **登入使用者**，我希望 **能查詢過去的抽號結果紀錄**，以便 **在需要時回顧或作證**。
- 作為 **活動主辦方 (使用者)**，我希望 **能將抽出來的結果快速分享或匯出**，以便 **公佈在社群或群組中**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite 作為核心技術棧。
- **效能與安全考量**：
  - 確保隨機演算法具備足夠的隨機性與公平性。
  - 使用者密碼需經過加密（如 bcrypt）後才能存入資料庫。
  - 防止常見的 Web 攻擊（如 SQL Injection, XSS 等）。
  - 系統需能承受小規模活動（如 100 人內）的同時在線存取不崩潰。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 隨機抽取號碼（輸入範圍，產出數字）。
  - 讓使用者輸入自訂選項進行抽取。
  - 基本的使用者註冊與登入功能。
- **Should Have (應該有)**：
  - 登入使用者可查看歷史抽籤紀錄。
  - 儲存常用的自訂選項名單。
- **Nice to Have (可以有)**：
  - 抽籤過程的視覺動畫效果（增加期待感）。
  - 將抽籤結果產生專屬連結或圖片分享。
