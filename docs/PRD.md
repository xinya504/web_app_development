# 線上抽籤系統 PRD (產品需求文件)

## 1. 專案概述

### 背景與動機
在各類活動（如尾牙、班會、抽獎活動、分組等）中，經常需要公平、隨機且快速地從一組名單或項目中抽出特定對象。傳統紙本抽籤不僅耗時且不環保。本線上抽籤系統旨在提供一個簡單易用、可客製化的網頁應用，讓使用者能快速建立抽籤活動並即時獲得結果，確保流程的公開與公平性。

### 目標用戶
- **一般大眾**：需要進行抽籤活動的任何人（例如：活動主辦方、教師、社團幹部、一般朋友聚會等）。

### 核心價值主張
- **簡單快速**：無需繁瑣註冊，打開網頁即可手動輸入名單並開始抽籤。
- **公平透明**：使用隨機演算法，確保結果公平，並即時展示開獎畫面。
- **高靈活度**：支援自訂抽出數量，滿足單次單抽或單次多抽的情境。

---

## 2. 功能需求

### 主要功能

1. **建立抽籤活動**
   - **使用者故事**：作為一般大眾，我希望能夠建立一個獨立的抽籤活動名稱，以便區分不同的抽籤情境（如「尾牙抽獎」、「值日生抽籤」）。

2. **手動輸入名單**
   - **使用者故事**：作為活動主辦方，我希望可以手動輸入（或貼上）多行文字作為抽籤名單，以便快速建立候選項目清單。

3. **可設定抽出數量**
   - **使用者故事**：作為抽籤執行者，我希望能夠設定每次點擊抽籤時要抽出的人數或項目數，以便符合大獎一次抽一人、普獎一次抽多人的需求。

4. **即時顯示結果**
   - **使用者故事**：作為參與者，我希望系統能在抽籤後即時以清晰、有視覺效果的方式顯示中獎結果，以便增加抽籤的趣味與期待感。

5. **不重複抽籤（抽出項目自動排除）**
   - **使用者故事**：作為活動主辦方，我希望被抽中的項目在接下來的抽籤中會自動被排除，以便確保同一個人或選項不會被重複抽出。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架。
- **前端渲染**：使用 Jinja2 模板引擎搭配純 CSS/JS 打造具備動態和美觀效果的使用者介面。
- **資料庫**：使用 SQLite，便於輕量化部署與儲存活動資料。

### 效能與安全考量
- **效能**：系統需保證即便名單有數百項，點擊抽籤後的隨機計算與結果顯示能在 1 秒內完成。
- **安全**：需防範基本的 XSS 攻擊，確保使用者輸入的名單或活動名稱在畫面上渲染時經過適當的跳脫處理（Sanitization）。

---

## 4. MVP 範圍

### Must Have (必須具備)
- 可輸入活動名稱並建立抽籤頁面。
- 提供大文字框讓使用者以換行分隔手動輸入多個名單項目。
- 可設定本次抽籤要抽出多少個項目。
- 點擊「抽籤」按鈕後，以隨機方式選出結果並即時呈現於畫面上。

### Should Have (應該具備)
- 不重複抽籤功能：系統記錄已抽出的名單，避免後續抽籤重複抽中。
- 顯示剩餘未抽名單數量。

### Nice to Have (有了更好)
- 視覺動態效果：輪盤動畫或名單快速捲動的搖獎特效。
- 歷史記錄：列出當前活動所有已抽出的結果與批次紀錄。
- 匯出功能：將最終所有中獎結果匯出為 CSV 檔以便後續作業處理。
